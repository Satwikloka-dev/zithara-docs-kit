## Customer (Lead) Integration

### What is a Customer API?

You use this to create, update, or read customer details from Zithara.

## 1. Create / Update Customer

```http
POST /v1/customer
```
### Use Case

- Send customer data from your system to Zithara.
- If the customer exists (based on phone or email), their record is updated.
- If not, a new one is created.

---

### Analogy

Like filling out and submitting a customer form — Zithara either adds a new record or updates the existing one.

---

### Authentication

You need an access token (see Access Token section).

---

### Payload

Full payload is available in the Postman Collection.

---

### Response

- **201 Created** – when a new customer is added  
- **200 OK** – when an existing customer is updated


## 2. Get Customer Details

```http
GET /v1/customer
```

### Use Case

Get customer details by providing one of the following:

* Phone
* Email
* Zithara Customer ID
* Platform Customer ID

---

### Example

```http
GET /v1/customer?phone=+1234567890
```

---

### Response

Customer profile in detail.

More information and response format are available in the
